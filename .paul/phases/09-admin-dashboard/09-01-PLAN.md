# Phase 09 — Admin Dashboard

## Goal
Give GBA staff/admins a dedicated panel to manage tutor requests (accept/decline) and view the full user roster.

## Deliverables

### Supabase (manual: run in SQL editor)
- `supabase/migrations/007_admin_role.sql`
  - `is_admin(user_id uuid)` SECURITY DEFINER helper function
  - Widens `profiles.role` check to include `'admin'`
  - `admin_select_all_profiles` policy — admin can SELECT all profiles
  - `admin_select_all_requests` policy — admin can SELECT all tutor_requests
  - `admin_update_requests` policy — admin can UPDATE tutor_request status

### Code Changes
- `types/index.ts` — added `'admin'` to Profile.role union
- `lib/admin.ts` — NEW: `getAllTutorRequests()`, `updateRequestStatus()`, `getAllUsers()`
- `app/admin.tsx` — NEW: admin dashboard screen (pushed from profile)
- `app/(tabs)/profile.tsx` — added "Admin Panel" entry (visible only when role === 'admin')

## Admin Screen UX
- Back button + "Admin Panel" title in custom header
- Red badge on header showing pending request count
- Segmented toggle: **Requests** | **Users**
- **Requests tab**:
  - Filter pills: Pending / All / Accepted / Declined (with counts)
  - Request cards: student → tutor, subject, status badge, days, message
  - Pending cards show Accept (green) / Decline (red) buttons
  - Optimistic UI update after accept/decline; Alert confirmation first
- **Users tab**:
  - Stats row: student count, tutor count, total
  - User rows: avatar initials, name, school, role pill

## Access Model
- Non-admin users who reach `/admin` see "Access Denied" screen
- Admin Panel entry in Profile is conditionally rendered (role === 'admin')
- To grant admin: set `role = 'admin'` on a user's profile row in Supabase
