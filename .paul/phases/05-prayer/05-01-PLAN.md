# Phase 05 — Prayer Requests

## Goal
Let students post prayer requests to a shared community board. Other students can react with "I'm Praying" to show support. Posts can be shared anonymously.

## DB Schema

```sql
prayer_requests (
  id          uuid PK default gen_random_uuid()
  user_id     uuid FK → profiles.id ON DELETE CASCADE
  body        text NOT NULL CHECK (length BETWEEN 10 AND 500)
  category    text ('personal','family','school','health','other') default 'personal'
  anonymous   boolean default false
  status      text ('pending','approved','rejected') default 'approved'
  created_at  timestamptz default now()
)

prayer_reactions (
  id          uuid PK default gen_random_uuid()
  request_id  uuid FK → prayer_requests.id ON DELETE CASCADE
  user_id     uuid FK → profiles.id ON DELETE CASCADE
  created_at  timestamptz default now()
  UNIQUE (request_id, user_id)
)
```

RLS: authenticated users can SELECT approved requests + all reactions; INSERT/DELETE own rows only.

## UI Features
- Navy header with "+ Request" pill button
- FlatList of prayer cards: category badge, relative timestamp, body excerpt, author (or "Anonymous"), "🙏 Pray / I'm Praying" toggle with count
- Optimistic pray toggle — updates instantly, reverts on network failure
- Pull-to-refresh
- Submit modal (pageSheet): body text input (10–500 chars with counter), category pill selector, anonymous toggle, submit button with loading state
- Character limit warning turns red above 450 chars
- Empty state with CTA to share first request
- Error state with retry
